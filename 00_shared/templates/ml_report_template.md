# ML 실험 노트 템플릿

> 이 템플릿은 결과 보고서라기보다 **실험을 복기하는 학습 노트**를 만들기 위한 형식이다.
> 숫자만 적지 말고, 읽는 사람이 “왜 이런 결과가 나왔는지”를 따라갈 수 있게 쓴다.

## 1. 실험 목적

- 이번 실험이 답하려는 질문
- 왜 지금 이 실험을 돌렸는지
- 연결 이론 문서 링크

## 2. 실험 전에 세운 가설

- baseline은 어디까지 갈 것 같은가
- strong baseline은 왜 더 좋아질 것 같은가
- 실패가 날 것 같은 구간은 어디인가

## 3. 실험 설정

- 데이터셋 / split 방식
- 전처리 방식
- 비교 모델 목록
- 실행 환경

## 4. 메트릭 읽는 법

- primary metric이 가장 중요한 이유
- secondary metric을 같이 봐야 하는 이유
- 이 실험에서 숫자를 오해하기 쉬운 지점

## 5. 결과 요약

- 모델 비교표
- 최고 모델
- baseline 대비 개선폭
- 한 줄 해석

## 6. 결과 figure 읽기

### `figure_name.svg`
- 링크
- 이 figure가 답하는 질문
- 실제 관찰값
- 해석

## 7. 분석 figure 읽기

### `figure_name.svg`
- 링크
- 무엇을 분석한 그림인지
- 실제 관찰값
- 가능한 원인
- 다음 가설

## 8. 실패 사례 분석

- 어디서 가장 많이 틀렸는가
- 어떤 slice가 특히 어려웠는가
- 데이터/모델 관점에서 원인을 어떻게 해석하는가

## 9. 이번 실험에서 배운 점

- 이론과 연결해서 새롭게 이해한 것
- baseline이 준 교훈
- strong baseline이 준 교훈

## 10. 다음 실험 가설

- 당장 이어서 해볼 실험 2~4개
- 무엇이 개선되면 성공이라고 볼지
