# ML 이론 노트 템플릿

> 이 템플릿은 “개념 요약”이 아니라 “공부 노트”를 쓰기 위한 뼈대다.
> 문장은 너무 딱딱한 보고서 말투보다, **왜 이런 질문이 생겼는지 → 어떤 용어를 알아야 하는지 → 실험에서 어디를 봐야 하는지**가 읽히도록 쓴다.

## 1. 이 stage를 왜 배우는가

- 이 stage가 없으면 다음 단계에서 어떤 실수를 반복하게 되는가?
- 이 문제를 배우면 무엇을 설명할 수 있게 되는가?
- 이전 단계와 비교해 새롭게 등장하는 질문은 무엇인가?

## 2. 먼저 알아야 할 용어

이 stage에서 처음 보는 용어를 짧게 풀어쓴다.

- 용어 A: 쉬운 말 설명
- 용어 B: 쉬운 말 설명
- 용어 C: 쉬운 말 설명

## 3. 문제 자체를 설명하기

- 입력은 무엇인가?
- 출력은 무엇인가?
- classification / regression / ranking / generation 중 무엇인가?
- 왜 단순한 방법만으로는 부족한가?

## 5. 메트릭을 설명하기

- primary metric은 무엇인가
- secondary metric은 왜 같이 보는가
- 각 metric이 답하는 질문은 무엇인가
- 이 metric을 오해하기 쉬운 지점은 무엇인가

## 6. 어떤 모델/기법을 쓰는가

- baseline 모델은 무엇이며 어디에 주로 쓰이는가
- strong baseline은 무엇이며 왜 강한가
- 딥러닝 비교군은 왜 넣는가
- 이번 stage에서 특히 주의해서 볼 모델은 무엇인가

## 7. 데이터셋을 설명하기

- 데이터셋 출처
- feature / target 구조
- 왜 이 stage에 맞는 데이터인가
- 분포나 누수 관점에서 주의할 점은 무엇인가

## 8. figure를 어떻게 읽는가

- 결과 figure 각각이 답하는 질문
- 분석 figure 각각이 답하는 질문
- 어떤 패턴이 보이면 좋은 신호인지
- 어떤 패턴이 보이면 다음 실험이 필요한지

## 9. 자주 하는 실수

- leakage 관련 실수
- metric 해석 실수
- baseline 없이 점수만 보는 실수
- figure를 만들고도 읽지 않는 실수

## 10. 다음 단계로 이어지는 질문

- 이 stage를 이해하면 다음 단계에서 무엇을 더 볼 수 있는가?
- 남아 있는 한계는 무엇인가?
- 다음 실험 가설은 어떤 방향이 자연스러운가?
