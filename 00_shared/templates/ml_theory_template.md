# ML Theory Note Template

> 이 템플릿은 "공부가 되도록" 이론 문서를 쓰기 위한 공통 형식이다.
> 단순 요약이 아니라, **왜 이 개념이 나왔는지 → 무엇을 해결하는지 → 수식/직관 → 실습 연결 → 자주 하는 실수** 흐름으로 작성한다.

## 1. 이 주제를 왜 배우는가

- 어떤 실무/실험 문제를 해결하려고 이 이론이 등장했는가?
- 이전 단계 접근법의 한계는 무엇이었는가?
- 이번 stage의 데이터/모델과 왜 연결되는가?

## 2. 배경 문제

- 문제 정의
- 입력/출력 형태
- 왜 단순한 방법만으로는 부족한가

## 3. 핵심 이론 설명

### 3-1. 개념 1
- 정의
- 직관
- 간단한 예시
- 필요하면 수식
- 이 개념이 없으면 어떤 문제가 생기는가

### 3-2. 개념 2
- 정의
- 직관
- 간단한 예시
- 필요하면 수식
- 이 개념이 없으면 어떤 문제가 생기는가

## 4. 메트릭 이론

- 이 stage에서 쓰는 핵심 metric은 무엇인가
- 그 metric이 답하는 질문은 무엇인가
- 다른 metric과 어떻게 다르고 언제 오해가 생기는가
- 현재 실험에서는 왜 이 metric을 우선으로 두는가

## 5. 실습 연결

- 이번 실험의 데이터셋에서 이 이론이 어떻게 드러나는가
- 전처리 / split / 모델 / figure 와 어떻게 연결되는가
- 실험 전에 어떤 가설을 세워야 하는가

## 6. 결과를 볼 때 체크할 질문

- 어떤 수치가 좋아져야 하는가?
- 어떤 figure 패턴이 좋아져야 하는가?
- 어떤 failure case가 줄어야 하는가?

## 7. 자주 하는 실수

- leakage 관련 실수
- metric 해석 실수
- figure 과잉해석 실수
- baseline 없이 강한 모델만 보는 실수

## 8. 다음 단계로 이어지는 질문

- 이 stage를 끝내면 다음에 무엇을 더 배워야 하는가?
- 다음 stage에서 이 개념이 어떻게 확장되는가?
